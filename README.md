# SpeckitTodoList

一個基於 Node.js、Express 與 SQLite 構建的現代化待辦清單系統，支援多使用者隔離、JWT 安全身份驗證與角色權限控管 (RBAC)。

## 🚀 快速啟動

### 1. 安裝依賴
```bash
npm install
```

## 🚀 快速啟動

### 1. 安裝依賴
在終端機執行以下指令，安裝專案執行所需的程式庫：
```bash
npm install
```

### 2. 設定環境變數 (重要)
「環境變數」是用來存放系統設定（如連接埠、密鑰）的地方。為了安全，這些設定不會存放在程式碼中，而是存放在一個名為 `.env` 的隱藏檔案裡。

**設定步驟：**
1. **複製範本**：在專案根目錄執行指令，建立屬於你的設定檔：
   ```bash
   cp .env.example .env
   ```
2. **產生金鑰**：執行以下指令產生一串隨機的「保險箱密碼」，並將產生的那一長串字串複製起來：
   ```bash
   node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
   ```
3. **編輯設定**：打開剛建立的 `.env` 檔案，將剛才複製的字串貼到 `JWT_SECRET=` 後面，並根據需要調整其他設定（詳見下方說明）。

## 🌐 環境變數詳解

你可以直接參考下方的範例來編寫你的 `.env` 檔案：

```env
# 伺服器設定
PORT=3000
NODE_ENV=development

# 安全金鑰 (請務必使用指令產生新的金鑰貼在這裡)
JWT_SECRET=3214ba8554664489a076e72f21df39994500ee18d299de46c8ffea10112341dab91eb20b6f0698b9fddc48d2d270a35e47243a861c2f2a1a8e7ff2d682bb6f92

# 管理員與資料庫
ADMIN_PASSWORD=你的安全密碼
DB_PATH=database.sqlite
```

以下是各個項目的詳細用途：

| 變數名稱 | 用途說明 | 建議設定值 | 為什麼這很重要？ |
| :--- | :--- | :--- | :--- |
| **`PORT`** | 伺服器的「門牌號碼」 | `3000` | 決定你在瀏覽器要輸入什麼網址（例如 `localhost:3000`）。 |
| **`JWT_SECRET`** | 登入憑證的「加密印章」 | *使用指令產生* | **極度重要！** 這決定了你的登入狀態是否安全，絕對不能洩漏給他人。 |
| **`ADMIN_PASSWORD`**| 管理員的「初始密碼」 | `你的自訂密碼` | 系統第一次啟動時，管理員帳號 (`admin`) 的登入密碼。 |
| **`DB_PATH`** | 資料庫檔案的名字 | `todo.sqlite` | 所有的待辦事項都會存在這個檔案裡。 |
| **`NODE_ENV`** | 執行模式 | `development` | 開發時設為 `development`；正式上線請改為 `production`。 |

### 🔒 安全小提醒
- **不要上傳 `.env`**：這個檔案包含敏感資訊，專案已經預設幫你過濾掉，請保持現狀。
- **定期檢查**：如果你懷疑金鑰洩漏，請隨時更換 `JWT_SECRET`（這會導致所有已登入的使用者強制登出）。

### 3. 啟動系統
本專案採前後端分離架構，建議開啟兩個終端機視窗分別執行：

#### A. 啟動後端伺服器 (Express + SQLite)
負責資料存取、身份驗證與 API 提供。
```bash
npm run dev:server
```
預設會執行於 `http://localhost:3000`。

#### B. 啟動前端開發伺服器 (Vite)
負責網頁渲染與即時預覽，並會將 API 請求轉發至後端。
```bash
npm run dev
```
啟動後請點擊終端機產生的連結（通常是 `http://localhost:5173`）即可進入系統。

## 🔑 使用者指南

### 身份驗證
- **註冊**：新使用者可建立帳號，預設角色為 `Viewer`。
- **登入**：使用憑據登入以存取個人待辦清單。
- **會話管理**：採用 JWT 並存於 Secure HttpOnly Cookie 中，有效期 24 小時。
- **暴力破解防護**：連續登入失敗 5 次將鎖定帳號 15 分鐘。

### 待辦清單管理
- **性能優化 (Performance Mode)**：針對低硬體規格（如 RAM <= 1GB）提供專屬優化模式，可移除昂貴視覺特效並提升操作流暢度。
- **多行輸入支援**：支援輸入多行描述，輸入框固定顯示 10 行高度並提供捲動條。
- **列表縮略**：在任務列表中，長篇內容將自動縮略顯示前 3 行，保持介面簡潔。
- **快捷鍵提交**：在輸入框中按下 `Ctrl + Enter` (或 Mac 的 `Cmd + Enter`) 即可快速提交任務。
- **資料隔離**：使用者僅能存取與操作屬於自己的任務。
- **看板模式**：支援「需求池」、「待辦」、「進行中」、「測試中」、「已完成」五個階段的任務管理。
- **即時編輯**：支援直接在清單中修改內容與狀態。

## 🛠️ 管理員介面 (Admin Dashboard)

系統為管理員提供專屬的控制台，用於監控系統狀態與管理使用者權限。

### 存取方式
1. 使用管理員帳號登入 (預設帳號：`admin` / 密碼：`admin`)。
2. 導覽至：`http://localhost:5173/pages/admin.html`

### 主要功能 (FR-011)
- **使用者清單**：即時查看所有已註冊使用者的帳號、角色、建立時間及鎖定狀態。
- **權限變更 (RBAC)**：支援將使用者角色調整為 `Admin`、`Editor` 或 `Viewer`。
- **手動解鎖**：若使用者因多次嘗試失敗而被鎖定，管理員可手動解除鎖定。
- **密碼重設**：管理員可協助使用者重設登入密碼。
- **安全日誌 (Audit Logs)**：查看系統安全相關事件 (如登入失敗、註冊、鎖定等)。
  - *註：日誌採自動清理政策，僅保留最近 10,000 筆紀錄。*

## 📂 目錄結構

- `server/`: 後端核心代碼 (Routes, Services, Middleware, DB)
- `services/`: 前端 API 封裝服務
- `public/`: 靜態資源 (HTML, CSS)
- `specs/`: 功能規格書、開發計畫與任務清單 (SDD 流程文件)
- `tests/`: 測試案例 (Unit, Integration)
- `logs/`: 系統執行與安全日誌

## 🛡️ 技術規範 (Constitution Highlights)
- **效能**：所有 API 平均回應延遲必須小於 200ms。
- **安全**：密碼 100% 以 bcrypt 加密儲存，所有輸入均進行 XSS 脫逸處理。
- **架構**：嚴格遵循 Service 層封裝邏輯，確保程式碼可測試性與維護性。
