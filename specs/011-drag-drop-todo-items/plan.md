# Implementation Plan: 拖拉式任務管理 (Drag-and-Drop Task Management)

**Branch**: `011-drag-drop-todo-items` | **Date**: 2026-05-11 | **Spec**: [specs/011-drag-drop-todo-items/spec.md](spec.md)
**Input**: Feature specification from `/specs/011-drag-drop-todo-items/spec.md`

## Summary

本功能旨在為 TODO List 引入直觀的拖拉式互動。使用者將能夠跨越不同的狀態河道（Lanes）移動任務卡片，並在河道內進行排序。技術上將採用成熟的拖拉函式庫（如 SortableJS）結合後端 SQLite 資料庫的 Rank 欄位持久化，確保操作流暢且資料一致。

## Technical Context

**Language/Version**: JavaScript (ES6+) / Node.js  
**Primary Dependencies**: Express (Backend), SortableJS (Recommended for Frontend Drag-and-Drop), Vite (Build Tool)  
**Storage**: SQLite (Backend), localStorage (Fallback/Client cache)  
**Testing**: Vitest, Supertest (Integration)  
**Target Platform**: Desktop Web (Chrome, Firefox, Safari)
**Project Type**: Web Application (Full-stack)  
**Performance Goals**: Drag interaction < 100ms response, API update < 200ms  
**Constraints**: < 200ms API latency, No external CDN (Local dependencies), Desktop-first  
**Scale/Scope**: Support up to 100 tasks per lane, 5-10 concurrent users (Last Write Wins)
